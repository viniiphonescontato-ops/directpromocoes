TAREFA: CRIAR A ABA JARVIS COMPLETA NO SAAS DIRECT PROMOCOES

Implemente esta funcionalidade dentro do projeto existente. Nao recrie o SaaS do zero e nao remova as telas, rotas, autenticacao, integracao Supabase ou funcionalidades atuais.

STACK OBRIGATORIA

- React
- TypeScript
- Tailwind CSS
- Framer Motion
- Three.js usando @react-three/fiber e @react-three/drei
- Lucide React para icones
- Supabase para dados, autenticacao e atualizacoes em tempo real

DEPENDENCIAS

Instale, caso ainda nao existam:

npm install three @react-three/fiber @react-three/drei framer-motion lucide-react
npm install -D @types/three

OBJETIVO

Adicionar no menu principal uma nova opcao chamada Jarvis. Ao clicar, abrir uma central exclusiva de inteligencia operacional. A experiencia deve ser dark, futurista, premium, poderosa, misteriosa e intimidadora, sem parecer um template generico.

Nao alterar o tema das outras telas. O modo Jarvis deve ser aplicado apenas na rota ou view Jarvis.

DADOS REAIS DO SAAS

O Jarvis deve ler os dados atuais do Supabase. Reutilize o cliente Supabase ja configurado. Nao use dados ficticios quando houver dados reais.

Tabelas existentes:

- demandas
- diaristas
- lojas_redes
- setores
- diarias_setor
- diarias_empresas

Mapeamento principal de demandas:

- id
- store_id
- date
- end_date
- sector
- spots
- worker_cost
- assigned_diarist_ids
- status

Calculos:

- Diarias solicitadas = vagas x quantidade de dias.
- Diarias atendidas = diaristas escalados x quantidade de dias, limitado ao solicitado.
- Pendencias = solicitadas - atendidas.
- Receita = diaria recebida da rede x vagas x dias.
- Custo = valor pago ao diarista x vagas x dias.
- Lucro = receita - custo.
- Loja atrasada = demanda com data final anterior a hoje e status diferente de concluida.

RESPOSTAS OPERACIONAIS

Implemente respostas imediatas para as sugestoes:

Resumo das demandas de hoje:
- Quantidade de demandas no periodo de hoje.
- Diarias solicitadas.
- Diarias cobertas.
- Posicoes sem escala.

Quem ainda nao fez check-in?:
- Caso ainda nao exista tabela de check-in, informar claramente que a analise usa diaristas escalados.
- Listar demandas com vagas sem cobertura.
- Preparar o codigo para futura tabela de check-ins.

Quais lojas estao atrasadas?:
- Listar lojas com demandas vencidas e nao concluidas.
- Exibir quantidade de demandas atrasadas.

Gerar relatorio executivo:
- Demandas cadastradas.
- Diaristas ativos.
- Faturamento previsto.
- Custo previsto.
- Lucro estimado.

Analisar operacao em tempo real:
- Demandas abertas.
- Diarias sem cobertura.
- Diaristas ativos ou em reserva.
- Principal risco operacional atual.

SUPABASE REALTIME

- Assinar mudancas nas tabelas demandas, diaristas e lojas_redes.
- Atualizar o snapshot do Jarvis quando houver INSERT, UPDATE ou DELETE.
- Remover os canais no cleanup do React.
- Exibir respostas usando o estado atualizado.

IA GENERATIVA OPCIONAL

Prepare o servico jarvisService.ts para chamar uma Supabase Edge Function chamada jarvis-assistant.

Regras de seguranca:

- Nunca colocar chave privada de IA no frontend.
- A Edge Function deve validar a sessao Supabase.
- Enviar apenas o snapshot operacional necessario.
- Aplicar limite de requisicoes.
- Nao enviar CPF completo, senhas ou outros dados sensiveis ao modelo.
- Se a Edge Function ainda nao existir, usar as respostas operacionais locais como fallback.
